# Práticas Scrum e XP

Este repositório é um template educacional para demonstrar, de forma prática e objetiva, como combinar Scrum e XP no desenvolvimento de software.

A proposta é mostrar como organizar trabalho, colaboração e qualidade técnica em ciclos curtos, com foco em entrega contínua de valor.

## Objetivo deste template

- Servir como base para estudos, workshops e disciplinas sobre métodos ágeis.
- Apoiar equipes iniciantes na adoção de práticas de gestão e engenharia.
- Demonstrar como cerimônias, artefatos e rotinas técnicas se complementam.

## O que é Scrum

Scrum é um framework de gerenciamento ágil para lidar com problemas complexos por meio de iterações curtas, chamadas sprints. O foco é criar incrementos frequentes de produto, com transparência e adaptação contínua.

### Pilares do Scrum

- Transparência: todos entendem prioridades, progresso e impedimentos.
- Inspeção: o trabalho e os resultados são avaliados frequentemente.
- Adaptação: ajustes são feitos rapidamente com base no aprendizado.

### Papéis do Scrum

- **Product Owner**: define e prioriza o que gera mais valor de negócio.
- **Scrum Master**: facilita o processo e remove impedimentos do time.
- **Developers**: constroem e validam o incremento de produto.

### Eventos do Scrum

- **Sprint**: ciclo de trabalho com duração fixa para gerar incremento.
- **Sprint Planning**: planejamento do objetivo e do que será desenvolvido.
- **Daily Scrum**: alinhamento diário sobre progresso e próximos passos.
- **Sprint Review**: apresentação do incremento e coleta de feedback.
- **Sprint Retrospective**: reflexão sobre processo e melhorias para a próxima sprint.

### Artefatos e compromissos do Scrum

- **Product Backlog**: lista priorizada de necessidades do produto.
- **Sprint Backlog**: plano de trabalho selecionado para a sprint atual.
- **Incremento**: resultado funcional entregue ao fim da sprint.
- **Product Goal**: direção de longo prazo do produto.
- **Sprint Goal**: objetivo central da sprint.
- **Definition of Done**: critérios de qualidade para considerar algo pronto.

## O que é XP (Extreme Programming)

**XP** é uma abordagem ágil com ênfase em excelência técnica, feedback rápido e colaboração intensa. Enquanto Scrum organiza o fluxo de trabalho e a cadência de entrega, XP fortalece como o software é construído com qualidade.

### Valores centrais do XP

- Comunicação
- Simplicidade
- Feedback
- Coragem
- Respeito

### Práticas comuns do XP

- **Histórias de usuário**: foco em necessidades do usuário final.
- **Desenvolvimento orientado a testes (TDD)**: testes guiam o desenho do código.
- **Programação em par**: duas pessoas colaboram na mesma tarefa crítica.
- **Refatoração contínua**: melhoria constante do design sem alterar comportamento.
- **Integração contínua**: integração frequente para reduzir conflitos e riscos.
- **Propriedade coletiva do código**: responsabilidade compartilhada sobre a base.
- **Pequenas entregas**: funcionalidades liberadas em incrementos curtos.
- **Ritmo sustentável**: manutenção de produtividade sem sobrecarga contínua.

## Como Scrum e XP se complementam

- Scrum responde principalmente ao que e quando entregar.
- XP reforça como entregar com qualidade técnica.
- Juntos, reduzem retrabalho, melhoram previsibilidade e aceleram aprendizado.

Na prática:

- Scrum fornece os rituais de alinhamento e priorização.
- XP fornece as práticas de engenharia para manter o código saudável.

## Execução do projeto

- Instalar dependências: `npm install`
- Executar aplicação: `npm start`
- Executar em modo desenvolvimento: `npm run start:dev`
- Rodar testes: `npm test`

> Observação: o template é intencionalmente simples para focar em práticas, não em complexidade técnica. Ele pode ser expandido com camadas de serviço, persistência e API conforme evolução do projeto.
